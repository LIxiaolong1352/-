## 经验教训（下次做App的前车之鉴）

### 1. 项目结构必须正确
- **错误**：文件上传到子文件夹（如"文件夹2"）而不是根目录
- **错误**：app文件夹内容散开，目录结构被破坏
- **正确**：直接上传完整项目文件夹，保持原有结构
- **建议**：先压缩成zip上传，再用workflow解压

### 2. GitHub Actions配置坑
- **错误**：缺少根目录的build.gradle和settings.gradle
- **错误**：gradlew文件权限问题（Windows上传后丢失执行权限）
- **错误**：缺少gradle.properties（AndroidX配置）
- **解决**：直接用gradle命令，不用gradlew
- **解决**：settings.gradle必须包含仓库配置（google(), mavenCentral()）

### 3. 在线构建平台选择
- **GitHub Actions**：配置复杂，容易出错，需要完整Android项目结构
- **Appcircle**：需要公司邮箱，个人开发者不方便
- **Codemagic**：✅ 推荐！支持个人GitHub账号，配置简单，构建快

### 4. 正确的Codemagic配置
```yaml
workflows:
  android-build:
    name: Build Android APK
    instance_type: mac_mini_m1
    max_build_duration: 10
    scripts:
      - name: Build APK
        script: |
          cd app
          gradle assembleDebug
    artifacts:
      - app/build/outputs/apk/debug/app-debug.apk
```

### 5. Android项目必备文件清单
- `app/build.gradle` - 应用级构建配置
- `app/src/main/AndroidManifest.xml` - 清单文件
- `app/src/main/java/.../*.java` - 源代码
- `app/src/main/res/` - 资源文件
- `build.gradle`（根目录）- 项目级构建配置
- `settings.gradle`（根目录）- 包含仓库配置
- `gradle.properties` - AndroidX配置

### 6. 测试建议
